# 統合アイデア案: TerraScore Life

## 一言でいうと

土地の質、災害リスク、子育て環境、地域の将来性をまとめて診断し、「この場所に住むべきか」を根拠付きで判断できる居住地デューデリジェンスAI。

## どの案をどう組み合わせたか


| 元案                 | 取り込む価値                | 統合後の役割          |
| ------------------ | --------------------- | --------------- |
| 案1: 土地環境スコアリングエンジン | 土地単位の環境品質評価           | 候補地の基礎コンディション診断 |
| 案3: 災害リスク統合ダッシュボード | 行政ハザード + 衛星由来の物理リスク   | 安全性とレジリエンス評価    |
| 案6: 子育て環境AIレポーター   | 学区、保育、医療、緑地など生活者視点    | 家族向けの暮らしやすさ評価   |
| 案8: 地域ポテンシャル診断AI   | 人口、交通、地価、都市計画などの中長期視点 | 将来価値と地域成長性の評価   |


## コンセプト

多くの人が家や土地を選ぶとき、見ている情報は分断されている。

- 不動産ポータルでは価格や間取りは見える
- ハザードマップでは災害リスクは見える
- 学区や保育園は別サイトで見る
- 地域の将来性や暑熱、緑地、地形までは統合されない

この分断を解消して、1つの地点またはエリアに対して以下の4視点を同時に返す。

1. 土地として健全か
2. 災害に対してどれだけ脆弱か
3. 子育て・生活のしやすさはどうか
4. 5年から10年で見た地域の伸びしろがあるか

## 誰のためのプロダクトか

### 第一ターゲット

住宅購入または転居を検討している子育て世帯

### 第二ターゲット

ファミリー向け物件を扱う不動産仲介、住宅デベロッパー

### 将来ターゲット

自治体の移住促進担当、地域金融機関、住宅ローンや保険の審査部門

## ユーザー課題

- 「住みやすい」の判断が口コミや印象に寄りがち
- 災害、安全、学区、緑地、暑さ、将来性を横並びで比較しづらい
- 物件単体の魅力と、地域全体のポテンシャルが分けてしか見えない
- 家族で意思決定するときに、根拠のある説明資料が作れない

## 提供価値

### 1. 今の住みやすさを見える化する

保育園、学校、医療機関、緑地、温熱環境をまとめて見せる。

### 2. 災害に対する不安を定量と文章で整理する

行政指定の災害区域だけでなく、標高、降水、地形などの物理条件も合わせて返す。

### 3. 将来の納得感を補強する

人口推計、交通、地価、都市計画から「この街が先細るか、維持されるか、伸びるか」を示す。

### 4. 家族の価値観に合わせた判断ができる

「保育園重視」「災害回避重視」「将来価値重視」など、家族ごとにレポートの見せ方を変えられる。

## プロダクトの体験

ユーザーは住所または地点を入力する。するとAIが JAXA と国交省のデータをまとめて取得し、次の4つのレイヤーでレポートを返す。

### レイヤー1: Land Quality

- 標高
- 地表面温度
- 緑地率
- 周辺の地形・地表コンディション

### レイヤー2: Risk & Resilience

- 土砂災害
- 急傾斜地
- 液状化
- 災害危険区域
- 大規模盛土
- 豪雨傾向や地形由来の注意点

### レイヤー3: Family Livability

- 小学校区、中学校区
- 保育園、幼稚園
- 医療機関
- 図書館、公民館
- 公園、自然環境
- 夏の暑さ、緑の多さ

### レイヤー4: Regional Potential

- 将来推計人口
- 駅利用状況
- 地価水準
- 都市計画、用途地域
- 立地適正化や開発余地のヒント

## 典型ユースケース

### ユースケース1: 子育て世帯の住宅購入比較

3つの候補地を比較して、「一番安心で、暮らしやすく、将来も納得できる場所」を選ぶ。

### ユースケース2: 不動産仲介の提案補助

営業担当が、物件紹介時に「なぜこのエリアが家族向きなのか」を出典付きで説明する。

### ユースケース3: 住宅デベロッパーの仕入れ判断

土地の物理条件と災害・生活利便・地域成長性をまとめて見て、ファミリー向け案件として成立するかを一次判断する。

## なぜこの組み合わせが強いか

案1と案3だけだと「安全で環境のよい土地評価ツール」に留まりやすい。案6を入れることで生活者の切実な意思決定に接続でき、案8を入れることで「今良い」だけでなく「将来も納得できる」説明になる。

つまり、

- 案1が「土地の質」を作り
- 案3が「不安の解消」を作り
- 案6が「生活実感」を作り
- 案8が「将来への納得」を作る

この4つが揃って初めて、住宅購入や転居の本当の意思決定に使える。

## MVPの切り方

広げすぎるとぼやけるため、MVPは「子育て世帯の候補地比較」に絞る。

### MVPで必須

- 1地点レポート生成
- 2〜3地点比較
- 4レイヤーの要約表示
- 出典付きの根拠表示
- 子育て世帯向けテンプレート

### MVPではやらない

- 投資利回り予測
- 自治体向けの政策シミュレーション
- リアルタイム災害アラート
- フルGISの地図編集機能

## データの役割分担

### JAXA

- 標高
- 降水
- 地表面温度
- 植生指数
- 衛星画像

役割は、行政データでは見えない物理環境の補完。

### 国交省 Geospatial MCP

- 学区
- 保育園・幼稚園
- 医療機関
- 地価
- 都市計画、用途地域
- 災害関連5系統
- 将来推計人口
- 駅乗降客数

役割は、生活・制度・将来性の説明材料。

### 国交省 DPF MCP

- PLATEAU
- 周辺インフラや関連データカタログ探索

役割は、拡張調査と補足データ導線。

## 収益化の方向性

### 短期

B2B2C。不動産仲介や住宅デベロッパーに SaaS 提供し、顧客への提案資料として使ってもらう。

### 中期

- 個人向け有料レポート
- 保険、住宅ローン、移住支援の周辺事業者への API 提供

## 差別化

- ハザードマップより広い。災害だけでなく暮らしと将来性まで扱う
- 不動産ポータルより深い。物件情報ではなく地域の本質を説明する
- GISより軽い。専門家向け操作ではなく意思決定向けレポートに寄せる
- 衛星データと行政データを同時に扱うため、「環境」と「生活」の両方に根拠を持てる

## このアイデアの勝ち筋

「土地を評価するAI」ではなく、「家族が住む場所を納得して決めるためのAI」として出すこと。

土地診断、災害診断、子育て診断、地域診断を別々の機能として売るのではなく、人生の大きな意思決定を支える1つの体験に束ねることで、4案の強みが最も自然に接続される。

## 次に仕様化するなら

1. 第一ターゲットを「未就学児のいる転居検討世帯」に固定する
2. 比較する地点数を 3 件までに制限する
3. 4レイヤーの表示項目と重み付けを仮決めする
4. 「断定」ではなく「判断支援」に徹する表現ルールを作る
5. 不動産仲介向けの提案書フォーマットを最初から意識する
